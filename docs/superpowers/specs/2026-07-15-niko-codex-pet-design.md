# Niko Codex Pet v2 设计规格

## 目标

基于用户提供的两张 Q 版角色参考图，为 Codex 制作一个现代像素风的 Niko 桌面宠物。最终资产必须保持角色身份和跨帧一致性，具备流畅的标准状态动画与完整的 16 向注视动画，并满足 Codex Pet v2 的打包与验证要求。

## 输入与视觉基准

- 身份参考：`C:\Users\29929\Downloads\ChatGPT Image 2026年7月15日 17_35_22.png`
- 身份参考：`C:\Users\29929\Downloads\ChatGPT Image 2026年7月15日 17_35_16.png`
- 已选风格概念：`C:\Users\29929\Desktop\codex_pets_Niko\concepts\niko-style-pixel.png`

两张原始参考图定义发型、面部、胡须、眼睛颜色、队服配色与整体人物气质。像素风概念图只定义渲染方向，不直接作为可打包 spritesheet，也不继承裤腿上的绿色旋涡。

## 角色身份锁定

角色必须在所有动画行中保持以下特征：

- 中等棕色侧分蓬松发型，前额有明显向一侧翻起的发束。
- 整齐的棕色全胡须、蓝色眼睛、浓眉与成人男性面部特征。
- 大头短身的 Q 版比例，紧凑、完整且易于在 `192x208` 单元格中辨认。
- 绿白短袖电竞队服，纹理简化为不可读的几何绿色块面；不生成文字、品牌名或精细队徽。
- 黑色运动长裤和白色运动鞋；裤腿不出现绿色旋涡、文字或额外图标。
- 基础形象不佩戴金牌。金牌只允许在 `waving` 庆祝动画中作为佩戴式、与身体相连的道具出现。

## 像素风规范

- 采用现代游戏像素画：清晰硬边、成组像素块、有限但层次充分的色板。
- 避免平滑矢量边、柔焦、绘画笔触、照片式皮肤、3D 材质和渐变背景。
- 发型、眉眼、胡须和鞋服的关键轮廓必须在应用显示尺寸下可读。
- 阴影仅用于角色内部体积，不生成投影、接触阴影、光晕或地面。
- 所有生成条带使用纯色 chroma-key 背景；最终图集由确定性流程转换为透明背景。

## 标准动画行

标准行按照 Codex Pet 的固定语义制作：

1. `idle`：轻微呼吸、一次眨眼和极小的头部起伏；循环不可静止。
2. `running-right`：面向屏幕右侧的交替步态，身体和四肢形成清晰节奏。
3. `running-left`：面向屏幕左侧的交替步态。仅当右跑翻转不会破坏服装与身份时，才从右跑安全镜像派生。
4. `waving`：一手挥动的庆祝姿态；此行允许佩戴金牌，但不允许波纹、星光或其他分离效果。
5. `jumping`：用身体高度表现起跳、腾空与落回，不生成地面、灰尘或冲击线。
6. `failed`：低头、肩部下沉或沮丧表情；不使用红叉、文字或漂浮符号。
7. `waiting`：期待用户输入的提问式姿态，必须明显区别于待机和审阅。
8. `running`：表示专注处理任务，通过眼神、头部和手部节奏表现；不得表现为脚步跑动。
9. `review`：通过前倾、眨眼、眉眼和头部小幅移动表现审阅，不新增纸张、放大镜或 UI。

每行独立生成并逐行验证。除安全的左右跑镜像外，不从其他状态复制或变换动画。

## 16 向注视机制

最终图集的第 9、10 行承载 16 个顺时针方向：

- 第 9 行：`000`、`022.5`、`045`、`067.5`、`090`、`112.5`、`135`、`157.5`。
- 第 10 行：`180`、`202.5`、`225`、`247.5`、`270`、`292.5`、`315`、`337.5`。

`000` 表示向上，`090` 表示屏幕右侧，`180` 表示向下，`270` 表示屏幕左侧。自然机制为：眼睛先引导，眼睑和眉毛配合，头颈随后小幅转动，上身只做克制的跟随；骨盆、双脚、身体基准和人物总体尺寸保持稳定。不得通过旋转、倾斜或拉伸整个人物来伪造方向。

先生成并批准四个基准方向，再分别生成两条连贯的八姿态方向行。相邻方向必须以均匀的 `22.5` 度视觉步进过渡，头部、面部、发型和上身不能跳动或突然改变比例。

## 资产流水线

1. 用 `prepare_pet_run.py` 创建请求、提示词、布局参考和视觉任务清单。
2. 用 `imagegen` 生成规范化的像素风主参考图，作为所有动画行的身份源。
3. 逐行生成并立即提取、检查标准动作；失败时只重做最小的完整动画行。
4. 生成四方向基准，批准方向语义后生成第 9 行，再验证并生成第 10 行。
5. 确定性提取帧、统一比例与基线、组装标准图集和 v2 扩展图集。
6. 对最终 v2 图集执行一次边缘去色、格式验证、动画预览和方向 QA。
7. 将通过验证的 `spritesheet.webp` 与 `pet.json` 一起暂存到 Codex 自定义宠物目录。

## 失败处理

- 身份、风格、服装或脸部漂移：重做对应完整动画行。
- 动作语义错误、方向相反、步态不交替或循环静止：重做对应完整动画行。
- 单个方向错误：加强方向提示并重做其所属的完整八姿态方向行，不拼补单独格。
- 提取造成的尺寸跳动但原条带稳定：改用稳定槽位提取，不重新生成视觉素材。
- 裁切、重叠、透明内部裂缝、分离效果或基准方向歧义：阻止打包，直到修复通过。
- 最终 chroma 去色或 v2 验证失败：作为确定性流水线错误处理，不通过重复生成图片掩盖。

## 验证与验收

最终交付需同时满足：

- 图集为 PNG 或 WebP，尺寸严格为 `1536x2288`，单元格为 `192x208`。
- `pet.json` 包含 `spriteVersionNumber: 2`。
- 九个标准动画行身份一致、动作可辨、循环流畅，无尺寸跳动和错误朝向。
- 四个基准注视方向明确，16 个方向形成连续顺时针循环，无反转或整身旋转。
- 三次隔离盲测的基准方向通过严格多数判断，所有方向具有显式语义结果。
- 最终透明背景、已使用单元格非空、未使用单元格全透明，且 v2 验证和 chroma 去色报告通过。
- 生成标准与扩展接触表、逐行动画 GIF、方向检查图、连续性报告和运行摘要。
